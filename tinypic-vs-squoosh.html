Tiny Pic vs Squoosh

两个工具都能在浏览器里压缩图片、不上传服务器。但侧重点不同：Squoosh 是 Google 的实验性 PWA，主打单图实时对比和探索新格式；Tiny Pic 是 HolaTiger 旗下的批处理工具，主打一次压多张 + ZIP 打包 + 零后端部署。

功能对比

选哪个

单张图调参 + 探索格式 → 选 Squoosh，它的实时滑块对比是同类最强。
一次处理 10-100 张 → 选 Tiny Pic，批量 + ZIP 打包是刚需。
需要部署到自己的服务器 → 选 Tiny Pic，单页 + 静态资源即可。

共同点

  • 都基于 WebAssembly 在浏览器内编码，文件不上传
  • 都支持 JPEG / PNG / WebP / AVIF（AVIF 在 Squoosh 是实验性）
  • 都不需要登录、不留账号

常见问题

Tiny Pic 和 Squoosh 哪个更好？

取决于场景。单图实时对比和探索 WebP/AVIF 等新格式选 Squoosh；批量处理、ZIP 打包、纯前端零依赖选 Tiny Pic。

Squoosh 是 Google 的项目吗？

Squoosh 是 GoogleChromeLabs 推出的开源 PWA，基于 WebAssembly 和 libimagequant / MozJPEG 等编码器。

Tiny Pic 支持哪些 Squoosh 不支持的功能？

批量上传并打包 ZIP、目标文件大小自动寻优、保留 EXIF 控制、完全无外部依赖（自托管友好）。
